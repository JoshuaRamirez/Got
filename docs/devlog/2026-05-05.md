## 20:24 UTC — P2 design decisions resolved on `claude/p2-design-decisions`

Cut `claude/p2-design-decisions` from `main` and executed every item from the
root `CLAUDE.md`'s "P2: Open Design Questions" list.

**Rules now codified in `docs/design-rules.md`** (root `CLAUDE.md` slimmed to a
pointer):

1. `context.Context` is the first parameter of every `Engine`/`Service`
   interface method. It is *not* added to pure value-type accessors
   (`Graph`/`Subgraph`/`Frontier`/`View`/...), to `identity.Hasher`/`Factory`
   (deterministic compute), or to `governance.Policy.Check` (Policy is a value,
   not an Engine). `namespace.Store` is the named exception — it gets `ctx` on
   every method because it is the only stateful interface and may be backed by
   a remote store.

2. Sentinel errors are declared at package scope and wrapped via
   `fmt.Errorf("%w: ...", ErrX, detail)`. Added across the codebase:
   `governance.ErrPolicyViolation`, `composition.ErrConflictUnresolvable`,
   `composition.ErrNoPushout`, `verification.ErrCertificationFailed`,
   `verification.ErrEnvironmentMismatch`, `revision.ErrSideConditionFailed`,
   `revision.ErrNoMatch`, `realization.ErrTargetUnsupported`,
   `release.ErrPolicyGate`, `release.ErrUnknownVersion`,
   `replay.ErrNonDeterministic`, `repo.ErrIngestRejected`,
   `temporal.ErrUnknownVertex`, `multiagent.ErrNoAuthorship`,
   `capability.ErrNoEmergence`, `projection.ErrInvalidSelector`,
   `provenance.ErrUnknownVertex`. The pre-existing `graph.Err*` set is kept.

3. Struct-vs-interface rule: single-getter "data holder" types are structs.
   Interfaces are reserved for multi-method contracts, opaque computation
   results, or types with real implementation alternatives. Converted to
   structs: `provenance.Trace`, `replay.Outcome`, `realization.FidelityContract`,
   `capability.Witness`, `composition.MergeWitness`, `multiagent.Responsibility`.
   Multi-method interfaces (`composition.Conflict`, `verification.Certificate`,
   `verification.Evaluation`, etc.) stay as interfaces.

4. `repo.Service.Ingest(State, any)` replaced with
   `Ingest(ctx context.Context, s State, p Payload)`. `Payload` is a typed
   interface with a `PayloadKind() string` discriminator that mirrors
   `graph.Query`.

5. Test gating: every package now has a `*_test.go`. The five concrete-impl
   packages (`graph`, `identity`, `namespace`, `ontology`, `provenance`) keep
   their behavioral tests; the twelve interface-only packages now have minimum
   tests covering exported constants, struct round-trips, and sentinel
   identity. Behavioral tests for those will land with the first concrete
   implementations per the design-rules doc.

**Implementation notes**:

**Layout**:
- `docs/use-cases/CLAUDE.md` — convention: ID scheme (`UC-U<NN>` for sea
  level, `UC-S<NN>` for fish level), template usage, writing rules.
- `docs/use-cases/template.md` — the Cockburn template every UC follows.
- `docs/use-cases/index.md` — catalogue index, two tables (user, system).

- `docs/use-cases/system/UC-S01..UC-S20.md` — 20 sub-function operations.

**Cockburn shape per file**: header table (goal level, scope, primary
actor, stakeholders & interests, preconditions, trigger, success/failure
postconditions); main success scenario as numbered steps; extensions
split into successful variations and failure paths (with sentinel error
references); sub-variations; related use cases (includes / extended-by /
extends).

**Coverage**: every public method on every `Engine` and `Service` in
`internal/` is reachable from at least one user UC and exercised by at
least one system UC. Failure paths name the specific package sentinel
(`graph.ErrVertexNotFound`, `revision.ErrNoMatch`,
`composition.ErrNoPushout`, etc.). The catalogue is now part of the
design surface — out-of-date entries are bugs, per the rule recorded in
`docs/use-cases/CLAUDE.md`.

`docs/CLAUDE.md` updated to point at the catalogue and add use-case
locations to the "where things go" table.
