## 15:35 UTC тАФ projection.EditedFrontier.Clone

Added `Clone()` to `*projection.EditedFrontier`. Closes the
"frontier mutation invalidates reuse" pitfall I flagged at the bottom
of the composition bridge analysis (see 2026-05-21).

### What it does

`Clone()` returns an independent copy of the frontier:

- `IDs` slice copied.
- `Vertices` and `Edges` maps copied.
- Each Vertex's and Edge's `Attrs` map is shallow-cloned so mutating
  `clone.Vertices[id].Attrs[k]` does not affect the original.

Reference-typed values inside an `Attrs` map (nested maps, slices) are
not deep-cloned. Documented as such тАФ callers who store such values are
responsible for any deeper cloning they need.

### Why this was the next-best move

Of the five forward directions listed in 2026-05-21:

1. `Clone()` (this PR)
2. More stock resolvers
3. Symmetric Strict bridge for `revision.Apply`
4. Domain-aware `Attrs` equivalence
5. Three-way `MergeThreeWay(ancestor, left, right)`

This was the cheapest (~50 LOC including tests), addressed a real
documented pitfall, and didn't open speculative scope. The remaining
four all require a user need to motivate them тАФ there are no active
callers asking yet. Stopping here preserves the "bridge functionally
complete" status without churning on features the project doesn't
need yet.
